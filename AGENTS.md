# AGENTS

- To get context about the project before making changes, use the CodeGraph index: a `.codegraph/` directory exists at the repo root. Prefer the `codegraph_explore` tool (or `codegraph explore "<symbols>"` from the shell) over `grep`/`find`/`Read` loops when you need to locate code, understand a flow, or see the call path between symbols. It returns the verbatim source plus the call path in one capped call.
- Stack is Python only. Project metadata lives in `pyproject.toml`; package root is `src/leaflet_automation`.
- Install/run entrypoint from `pyproject.toml`: CLI command is `leaflets`, implemented by `src/leaflet_automation/cli/main.py`.
- Verified syntax check command is `python3 -m compileall "src"`. There is no tracked test, lint, formatter, or typecheck config yet.
- Main runnable flows today are discovery AND extraction: `leaflets discover-lidl`, `leaflets extract-lidl <id>`, and `leaflets run-weekly-lidl` all work end-to-end.
- Current persistence is SQLite only. Schema is created at runtime by `src/leaflet_automation/db/schema.py` (tables: `leaflets`, `products`, `leaflet_pages`). Default DB path is `data/leaflets.db` from `src/leaflet_automation/app/config.py`.
- Runtime config uses `LEAFLETS_` environment variables via `pydantic-settings`. Verified defaults include `LEAFLETS_DATA_DIR=data`, `LEAFLETS_DATABASE_PATH=data/leaflets.db`, and `LEAFLETS_REQUEST_TIMEOUT_SECONDS=30.0`.
- Lidl code is adapter-based: shared interfaces are in `src/leaflet_automation/retailers/base.py`; Lidl-specific logic lives under `src/leaflet_automation/retailers/lidl/`.
- Discovery is seeded + dynamic. `src/leaflet_automation/retailers/lidl/discovery.py` starts from the public folhetos landing page and expands through `relatedFlyers`; `SEED_FLYER_IDS` is a hardcoded fallback when the page scrape fails.
- Product extraction IS implemented (`LidlAdapter.extract_products_from_page()` in `src/leaflet_automation/retailers/lidl/adapter.py`). OCR (`services/ocr.py`, RapidOCR), image block detection (`services/images.py`), screenshot cropping (`services/screenshots.py`), and OCR-assisted price matching all work today. A retailer-agnostic **projection-profile grid detector** also exists — `CardBox` + `CardDetector._grid_boxes()` in `src/leaflet_automation/services/cards.py` (US-1.1 ✅ DONE, PR #2) — but is **NOT yet wired into the adapter** (wiring is US-3.1). The only remaining stub is PDF extraction (`services/pdf.py`). See `docs/IMPLEMENTED_FEATURES.md` for the full inventory of what works.
- Known extraction limitations being fixed by `docs/superpowers/specs/2026-07-08-generic-card-extraction-design.md`: (a) `services/images.py:10 detect_product_blocks()` slices pages into equal vertical strips, so screenshots span multiple adjacent products — the grid detector (`services/cards.py`) is the replacement but is not wired in yet (US-3.1); on dense real pages (e.g. `page-04.png`) the grid-only path saturates to full-width strips, which the US-1.2 contour fallback and US-5.2 per-card regression will address; (b) ~~`adapter.py:76` skips the keyword-name fallback when alt-text yields any names, dropping visible products~~ **FIXED by US-4.1** — keyword fallback now runs in union with alt-text, with case-insensitive de-dup (`adapter.py` `extract_products_from_page`, `seen_names` set); tests in `tests/test_us_4_1_keyword_union.py`. The current work scope is "fruits" completeness — every fruit card on the latest leaflet's produce pages must be extracted, none dropped.
- Candidate Lidl pages are currently filtered only by `altText`/`keyWords` keyword matching in `src/leaflet_automation/retailers/lidl/filters.py`; this is the category gate for `Frutas & Legumes` work.
- Dependencies already installed in the venv but NOT declared in `pyproject.toml`: `opencv-python` and `numpy`. Declare `opencv-python>=4.8` when the card-detection work lands.
- Key docs: `docs/LIDL_LEAFLETS_API_RESEARCH.md` (read it before changing discovery, date filtering, or API usage), `docs/IMPLEMENTED_FEATURES.md` (what's implemented), `docs/IMPLEMENTATION_BACKLOG.md` (remaining work), `docs/superpowers/specs/2026-07-08-generic-card-extraction-design.md` (the extraction-fix design).
